📞 Telco Customer Churn Prediction with GCP & Vertex AI MLOps + GCP
🐍 Python ☁️ BigQuery ☁️ Cloud Storage ☁️ Vertex AI 🤖 XGBoost 📊 Looker Studio 📈 Scikit-learn 🐘 PostgreSQL

🔥 Masalah Bisnis: Operator telekomunikasi kehilangan pelanggan secara signifikan karena tidak dapat mengidentifikasi churn lebih awal. Network Quality of Service (throughput, latency, drop rate) tidak terintegrasi dengan data pelanggan, sehingga tim bisnis kesulitan mengambil tindakan preventif.

🛠️ Solusi & Tindakan: Saya membangun end-to-end MLOps pipeline di Google Cloud Platform (GCP): data ingestion di Cloud Storage, pemrosesan di Dataflow (Apache Beam), penyimpanan data bersih di BigQuery, dan pelatihan model XGBoost di Vertex AI. Mengintegrasikan Network KQI sebagai fitur prediktif, menghasilkan AUC 87.65% dan recall 77.1%.

📊 Dampak Bisnis: Model terdaftar di Vertex AI Model Registry dan terhubung ke Looker Studio dashboard untuk visualisasi real-time. Contract type dan Network Quality Score menjadi pendorong utama churn—memungkinkan tim untuk mengambil tindakan preventif secara proaktif dan meningkatkan retensi pelanggan.

📡 Telecom Milan Network Traffic Analysis END-TO-END DS PIPELINE
🐍 Python 🔄 ETL 2.5GB 🎯 K-Means 📊 Time Series 🤖 Gradient Boosting 🎨 Streamlit 🗺️ Spatial Clustering 📈 R² = 0.802

🔥 Masalah Bisnis: Operator telekomunikasi di Milan menghadapi tantangan dalam memahami pola lalu lintas jaringan dari 14.3 juta record data (2.5GB). Tanpa pemahaman spasial dan temporal yang baik, mereka kesulitan mengoptimalkan kapasitas infrastruktur dan meningkatkan pengalaman pelanggan.

🛠️ Solusi & Tindakan: Saya membangun pipeline data science end-to-end: parsing data sparse, spatial clustering dengan K-Means untuk mengelompokkan area berdasarkan aktivitas, time series analysis, dan model prediktif Gradient Boosting (R²=0.802). Menemukan bahwa area residensial memiliki aktivitas 8x lebih tinggi daripada pusat kota.

📊 Dampak Bisnis: Memberikan peta aktivitas jaringan yang jelas untuk perencanaan kapasitas. Operator dapat mengalokasikan investasi infrastruktur secara tepat sasaran—misalnya menambah kapasitas di area residensial yang padat, dan mengoptimalkan biaya operasional di area dengan trafik rendah.

🗺️ ILASPP - Sistem Informasi Nilai Tanah Spasial Prediktif UPDATE
🐍 Python 📐 R 🐘 PostgreSQL + PostGIS 📊 Streamlit 🗺️ GWR 🌐 Kriging 🔥 Folium Heatmap

🔥 Masalah Bisnis: Penentuan nilai tanah oleh ATR/BPN seringkali kurang akurat karena tidak mempertimbangkan faktor spasial lokal. Akibatnya, potensi penerimaan pajak daerah tidak optimal dan masyarakat bisa dirugikan oleh penetapan nilai yang tidak sesuai.

🛠️ Solusi & Tindakan: Saya mengembangkan sistem prediktif berbasis Geographically Weighted Regression (GWR) dengan R² = 96.9% dan Kriging Interpolation untuk memprediksi nilai tanah di lokasi yang tidak terobservasi. Optimal bandwidth 186m memastikan model menangkap variasi spasial lokal secara akurat.

📊 Dampak Bisnis: Memberikan akurasi prediksi nilai tanah yang sangat tinggi untuk mendukung keputusan tata ruang dan perpajakan. Dashboard interaktif dengan Folium heatmap memudahkan petugas BPN memvisualisasikan dan memvalidasi prediksi di lapangan.

🚀 Grab Expansion Priority Analysis STRATEGIC PLANNING
🐍 Python 📊 Pandas 🎨 Streamlit 📈 Plotly 🗺️ Folium 📐 Matplotlib 🐘 PostgreSQL

🔥 Masalah Bisnis: Grab ingin memperluas layanan GrabFood & GrabCar ke 35 kabupaten/kota di Jawa Tengah, namun tim ekspansi kesulitan menentukan prioritas karena data tersebar dan tidak terstruktur. Tanpa prioritas yang jelas, investasi pemasaran dan logistik bisa terbuang sia-sia di area dengan potensi rendah.
